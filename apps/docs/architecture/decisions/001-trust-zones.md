# ADR 001: Separate privileged and preview trust zones

Status: Accepted

## Context

The main application can control devices, files, ports, processes, and inspection sessions. Preview content may be generated by or connected to an inspected application and cannot receive the same authority safely.

## Decision

The privileged application WebView is never remotely debuggable in production. Preview content runs in a separately labeled window or WebView with a dedicated least-privilege capability. Production CSP is explicit. Development debugging requires an explicit build-profile gate and cannot use wildcard remote origins.

## Consequences

Preview features must communicate through narrow validated commands. Adding a new preview capability requires a security review and regression test. Disabling a preview is the safe rollback; restoring broad production permissions is not.
