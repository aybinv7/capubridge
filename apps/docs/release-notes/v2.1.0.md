# CapuBridge v2.1.0

Small follow-up to v2.0.0 — documentation and tool-description improvements only, no behavior
or capability changes.

## AI / MCP tool descriptions

Live use of the v2.0.0 MCP tools surfaced two things worth documenting up front instead of
being discovered by trial and error:

- **Foreground requirement.** `evaluate_js`, `click_element`, `read_storage`, `read_console`,
  and `read_network` all need the target app in the foreground — Android can freeze a
  backgrounded app's entire process, which silently hangs any CDP call to it. There's no
  reliable automatic fix for this (the only override is a device-wide `adb` setting plus a
  reboot), so the tool descriptions and the server's top-level instructions now say clearly:
  bring the app forward with `launch_app` first, and re-check `list_targets` after switching
  apps, since targets can go stale.
- **Capacitor SQLite pattern.** `read_storage` doesn't have a dedicated mode for a Capacitor
  app's own SQLite database (`@capacitor-community/sqlite`) yet, but `evaluate_js` can reach
  it fine via `window.Capacitor.Plugins.CapacitorSQLite.query(...)` — as long as the
  `database` argument is the plugin's _logical_ name (e.g. `presalio`), not the on-disk
  filename (e.g. `presalioSQLite.db`), which silently returns empty results instead of
  erroring. This is now documented on `evaluate_js` and in the server's connection
  instructions; a corrected design-doc note replaces an earlier, wrong claim that this
  wasn't possible at all.

No new tools, no changed behavior — every existing test still passes.
